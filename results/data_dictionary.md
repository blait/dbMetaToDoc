# Data Dictionary — schema `cdm`

**Domain:** healthcare / clinical research (observational health data)

This database is a full implementation of the OHDSI OMOP Common Data Model (CDM) version 5.3.1, used to standardize observational patient-level health data for analytics and research. Per its cdm_source record, it holds a Synthea-generated synthetic patient population. The schema includes the complete set of OMOP CDM components: standardized clinical event tables (person, observation_period, visit_occurrence, condition_occurrence, procedure_occurrence, drug_exposure, measurement, observation, device_exposure, specimen, death, note), derived "era" tables (condition_era, drug_era, dose_era), health economics tables (cost, payer_plan_period), the standardized vocabulary/terminology tables (concept, concept_relationship, concept_ancestor, concept_synonym, concept_class, vocabulary, domain, relationship, drug_strength, source_to_concept_map), and metadata/derived-element tables (cdm_source, metadata, cohort_definition, attribute_definition, fact_relationship, location, care_site, provider). In practice the instance is partially populated: it contains 2,694 persons with associated visits (1,037 inpatient encounters), 65,719 condition eras, 52,508 drug eras, ~67,707 drug exposures, 37,409 procedures, 1,477 observations, and measurement records, all backed by the standard vocabularies (RxNorm, SNOMED, LOINC, etc.). Many optional tables — cost, payer/insurance coverage, devices, specimens, deaths, notes/NLP, locations, care sites, providers, and several vocabulary/cohort metadata tables — are empty, which is typical of a synthetic, vocabulary-anchored OMOP CDM dataset focused on core clinical events.
