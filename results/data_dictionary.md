# Data Dictionary — schema `cdm`

**Domain:** healthcare / clinical (OMOP CDM)

This is a healthcare research database implemented in the OMOP Common Data Model (CDM) version 5.3.1, populated from a Synthea synthetic patient population. It standardizes longitudinal clinical data for 2,694 patients (born 1909–1985) across the full set of OMOP clinical event tables — person demographics, observation periods, visit occurrences (all coded as inpatient hospitalizations), condition occurrences and eras, drug exposures and eras, procedures, measurements, observations, and deaths — spanning records from roughly 1910 to 2019. It includes the standardized OMOP vocabulary tables (concept, concept_relationship, concept_ancestor, concept_synonym, vocabulary, domain, relationship, drug_strength) that map and hierarchically organize standardized clinical concepts drawn from terminologies such as SNOMED, RxNorm, LOINC, ICD9CM, and UCUM, plus health-system reference tables (care_site, provider, location). Several optional/derived tables (cost, payer_plan_period, attribute_definition, concept_class, metadata, fact_relationship) are present but empty. The database's purpose is to support standardized analytics and observational research on synthetic patient health records.
